= AFX 儲存系統 SVM 的附加管理
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
除了典型的 AFX SVM 管理之外，您可能還需要根據您的環境執行其他任務。大多數附加任務都可以使用系統管理員執行，但在某些情況下您可能需要使用 CLI。


NOTE: 所描述的ONTAP功能和管理對於運行 Unified ONTAP 的AFX 儲存系統和AFF或FAS系統來說是通用的。其中包含相關 Unified ONTAP文件的連結（視情況而定）。



[[storage-management-and-performance]]
== 儲存管理和效能

您可以在 AFX 部署中設定和使用多種可選的儲存管理和效能功能。



[[ontap-nas-storage-management]]
=== ONTAP NAS 儲存管理

網路附加儲存（NAS）提供專用檔案儲存空間，可供網路中的多個用戶端存取。ONTAP 支援多種 NAS 協定。深入瞭解 https://docs.netapp.com/us-en/ontap/nas-management/index.html["NAS儲存管理"^]。



[[ontap-flexcache-volumes]]
=== ONTAP FlexCache卷

FlexCache 是 ONTAP 的一項遠端快取功能。它將資料更靠近用戶端，從而提高存取效能並降低成本。建立一個 FlexCache Volume，該 Volume 最初僅從來源檔案系統複製 Metadata，從而簡化檔案分發並減少 WAN 流量。深入瞭解 https://docs.netapp.com/us-en/ontap/flexcache/index.html["ONTAP FlexCache卷"^]。



[[ontap-flexgroup-volumes]]
=== ONTAP FlexGroup卷

FlexGroup Volume 由多個成員 Volume 組成，這些成員 Volume 可以自動透明地共享流量。FlexGroup Volume 具有諸多優勢、包括高效能和簡化的管理。深入瞭解 https://docs.netapp.com/us-en/ontap/flexgroup/creation-workflow-task.html["FlexGroup卷設定"^]。



[[cross-file-sequential-read]]
=== 跨檔案循序讀取

跨檔案順序讀取（CFSR）是 ONTAP 9.19.1 及更高版本 AFX 系統中的一項功能，可提升影片編輯及相關應用程式的效能。對於將視訊幀儲存為單獨的順序檔案（通常每個檔案大小為 50 至 250 MB）的工作流程，CFSR 可根據順序檔案名稱模式智慧預測並預先擷取下一個檔案。最多可預先擷取 10 個檔案或 500 MB 的資料，從而顯著降低讀取延遲，並確保以每秒 30 幀的速度流暢播放 4K/8K 原始影片。

在執行 ONTAP 9.19.1 或更新版本的 AFX 系統上，CFSR 預設為啟用，但您需要在個別磁碟區上明確啟用。儲存管理員可以使用 ONTAP CLI 或同等的 REST API 呼叫來設定此功能。使用 CLI 時，您可以在建立新磁碟區的過程中啟用 CFSR。您也可以使用下列命令更新現有的磁碟區：

`volume modify -volume <volume_name> -aggressive-readahead-mode [cross_file_sequential_read|none]`

.範例
[source, bash]
----
volume modify -volume myvolume -aggressive-readahead-mode cross_file_sequential_read
volume show -volume myvolume -fields aggressive-readahead-mode
----
CFSR 功能與 FlexVol 和 FlexGroup Volume 相容，且與檔案副檔名無關，支援十進位和十六進位檔案名稱編號方案。對於 NFSv4.x 傳輸協定，CFSR 與多傳輸協定 I/O（MPIO）讀取功能整合，透過直接從記憶體提供後續讀取來進一步提升效能。CFSR 僅適用於 AFX 系統，且對支援的 Volume 有所限制。下列 Volume 類型不支援 CFSR：

* FabricPool 啟用的磁碟區
* FlexCache 端點磁碟區
* SnapMirror 目的地磁碟區


CFSR 不支援任何其他 ONTAP 平台（FAS、AFF 或 ASA）或雲端部署，例如 Cloud Volumes ONTAP。它可以在啟用或停用 GDD 的 FlexGroup 磁碟區上運行。但是，由於視訊幀檔案通常遠小於 GDD 的最小分段大小，GDD 無法有效地將這些檔案條帶化到各個 FlexGroup 組成部分。這意味著 GDD 對 CFSR 的檔案預取行為幾乎沒有直接影響。



[[data-protection]]
== 資料保護

您可以在 AFX 部署中設定和使用多種可選的資料保護功能。



[[consistency-groups]]
=== 一致性組

一致性群組是作為單一單元管理的磁碟區集合。深入瞭解 https://docs.netapp.com/us-en/ontap/consistency-groups/index.html["ONTAP 一致性群組"^]。



[[snaplock]]
=== SnapLock

您可以將檔案轉換為磁碟區層級的一次寫入多次讀取（WORM）狀態來保護檔案。SnapLock 支援兩種模式。Compliance 模式確保檔案在保留期到期前無法刪除，從而滿足政府或行業特定要求。Enterprise 模式允許特權使用者在保留期到期前刪除檔案。深入瞭解 https://docs.netapp.com/us-en/ontap/snaplock/index.html["ONTAP SnapLock"^]。



[[snapmirror-active-sync-for-nas]]
=== 適用於 NAS 的 SnapMirror active sync

SnapMirror 主動同步功能可讓業務服務在發生完整站台故障時繼續運作。這項技術使應用程式能夠無縫地容錯移轉至備用複本，無需手動介入或自訂腳本。深入瞭解 https://docs.netapp.com/us-en/ontap/snapmirror-active-sync/protect-nas-task.html["適用於 NAS 的 SnapMirror active sync"^]。



[[storage-efficiency]]
== 儲存效率

ONTAP 提供多種儲存效率功能，可協助您最佳化儲存使用並降低成本。



[[global-deduplication]]
=== 全域重複資料刪除

https://docs.netapp.com/us-en/ontap/concepts/deduplication-concept.html["重複資料刪除"^] 是一項  https://docs.netapp.com/us-en/ontap/volumes/deduplication-data-compression-efficiency-concept.html["ONTAP 儲存效率"^] 功能，可透過識別和移除重複的資料區塊來降低實體儲存需求。傳統的重複資料刪除作業可在磁碟區或集合體層級進行。這項現有功能可為所有 ONTAP 平台的客戶提供顯著的儲存效率。

從 ONTAP 9.19.1 開始，AFX 引入了新的操作範圍，將重複資料刪除擴展到整個儲存可用區（SAZ）。全域重複資料刪除在磁碟區級重複資料刪除的基礎上，自動消除整個 SAZ 中所有磁碟區上的重複資料區塊。全域重複資料刪除工作階段會自動觸發，無需人工幹預即可確保最佳的節省效果。

AFF A1K 系統預設啟用全域重複資料刪除功能，該功能在背景透明運行，對效能的影響極小。無需功能授權，且該功能可與現有的儲存效率技術（包括壓縮和精簡）無縫整合。透過全域重複資料刪除所實現的節省將透過儲存效率指標進行報告。對於資料頻繁地跨多個磁碟區共享或複製的應用程式工作負載，此功能尤其有利。



[[security]]
== 安全

您可以在 AFX 部署中設定和使用多種可選的安全功能。



[[fpolicy]]
=== FPolicy

FPolicy 是一個檔案存取通知框架，用於監控和管理儲存虛擬機器（SVM）上的檔案存取事件。您可以使用 FPolicy 建立策略，根據您定義的條件，定義要監控的檔案操作，並可選擇封鎖這些操作。FPolicy 通常用於安全審計、合規性和資料治理。了解更多 https://docs.netapp.com/us-en/ontap/nas-audit/two-parts-fpolicy-solution-concept.html["ONTAP FPolicy 解決方案"^]。



[[nfs-over-tls]]
=== 基於 TLS 的 NFS

從 ONTAP 9.19.1 開始，ONTAP 為本地部署的 NFS over TLS 提供支援，並使用 TLS 1.3 對啟用 NFS 的 SVM 上的傳輸中 NFS 流量加密。此版本還支援可選的基於 LIF 的雙向 TLS 主機驗證，以及匯出原則控制，以拒絕相符用戶端的非 TLS TCP 存取。NFS over TLS 可與現有的 ONTAP NFS 基礎架構配合使用，因此應用程式可以繼續正常使用 NFS，同時 TLS 會對傳輸中的 NFS 流量進行加密。在 ONTAP 9.19.1 中，NFS over TLS 不適用於雲端平台。如需詳細資訊，請參閱 https://docs.netapp.com/us-en/ontap/nfs-admin/enable-nfs-over-tls-task.html["啟用 NFS over TLS"^]。
